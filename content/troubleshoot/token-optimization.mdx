## TL;DR — 5 招防止 API 費用失控

1. **開新 Session** — 每個任務用 `/new`，唔好讓 history 無限累積
2. **換平嘅模型** — 日常任務用 Haiku 或 DeepSeek，唔係每樣都要 Opus
3. **降低 Think Mode** — `think: low` 可以大幅減少 token 消耗
4. **用 Cron 代替持續監控** — 定時執行，唔好叫 Agent「不停 check」
5. **設用量上限** — 喺 API provider dashboard 設 hard limit 防意外

---

## 點解會咁貴？Session History 累積機制

MIT Technology Review 報導過有用戶用 Claude Opus 4.5 兩個鐘燒咗 **$100 美金**。唔係 Bug，而係 OpenClaw 嘅設計：**每次對話都會將整個 session 歷史傳畀 API。**

想像係咁：

```
第 1 次對話：傳 100 tokens（你嘅問題）
第 2 次對話：傳 100 + 500 tokens（問題 + 第 1 次完整對話）
第 3 次對話：傳 100 + 500 + 1200 tokens（問題 + 前兩次完整對話）
...
第 20 次對話：傳 幾萬 tokens
```

呢個係 LLM 嘅基本運作方式——AI 需要看到完整對話歷史先能理解 context。但問題係，好多人開咗一個 session 就用成日，越用越貴。

**我自己嘅情況：**

TUI 底部會實時顯示 token 用量，例如 `tokens 10k/272k (4%)`。跑住 5 個 Agent（main、crawler、webtest、coder、social），如果唔留意，一個長 session 可以輕易消耗幾萬 token。

<Callout variant="warning">
**最危險嘅組合：** 長 session + 頂級模型（Claude Opus 4.6 / GPT-5.4）。一個跑咗幾小時嘅 Coder Agent session，token 消耗可以係新 session 嘅 10–50 倍。
</Callout>

## 招一：定期開新 Session

最簡單、效果最大嘅一招。

**為什麼有效：** 新 session 冇歷史 context，每次對話只傳你當前嘅指令，token 消耗係最低點。

**3 種開新 Session 嘅方法：**

```bash
# 方法一：TUI 入面
/new

# 方法二：Telegram 打
/new

# 方法三：CLI
openclaw sessions new
```

**實際建議：**

- 每個獨立任務開一個新 session
- 完成一個項目後唔好繼續喺同一個 session 做下一件事
- Coder Agent 每次 code review 任務開新 session
- Crawler Agent 每次抓數據任務開新 session

<Callout variant="tip">
**記憶 vs Session：** 你嘅 Agent 嘅長期記憶同 session 係分開嘅。開新 session 唔會清除 Agent 嘅 persistent memory，只係重置當前對話的 context。
</Callout>

## 招二：揀啱模型（價格差距極大）

唔係所有任務都需要最貴嘅模型。用對模型可以慳 90% 費用。

### 主要 Provider 定價比較（2026 年 3 月）

| 模型 | Input（/1M tokens） | Output（/1M tokens） | 適合任務 |
|------|---------------------|----------------------|----------|
| Claude Opus 4.6 | $15.00 | $75.00 | 最複雜推理、架構設計 |
| Claude Sonnet 4.6 | $3.00 | $15.00 | 日常編程、分析 |
| Claude Haiku 4.5 | $0.25 | $1.25 | 簡單任務、高頻 Agent |
| GPT-5.4 | ~$10.00 | ~$30.00 | 通用、工具整合 |
| GPT-4o mini | $0.15 | $0.60 | 輕量任務、分類 |
| DeepSeek V3 | $0.27 | $1.10 | 高 CP 值，中文佳 |
| Kimi K2.5 | $0.15 | $0.60 | 中文任務、長 context |
| 本地模型（Ollama） | 免費 | 免費 | 需要 Apple Silicon / GPU |

*價格為參考，以各 provider 官方定價為準。*

**按任務揀模型：**

```yaml
# ~/.openclaw/config.yaml 範例
agents:
  coder:
    model: claude-sonnet-4-6      # 複雜代碼任務
  crawler:
    model: claude-haiku-4-5       # 簡單抓數據，高頻執行
  social:
    model: gpt-4o-mini            # 輕量文字處理
  main:
    model: claude-sonnet-4-6      # 日常主 Agent
    fallback_model: claude-haiku-4-5  # 超出 budget 時 fallback
```

## 招三：控制 Think Mode

OpenClaw 有三個 think mode，直接影響 token 消耗：

| Think Mode | Token 消耗 | 適合場景 |
|---|---|---|
| `think: high` | 最多（3–5x） | 複雜推理、架構決策 |
| `think: medium` | 中等（1.5–2x） | 一般任務 |
| `think: low` | 最少（1x） | 簡單指令、自動化任務 |

TUI 底部會顯示當前 think mode，例如 `think low`。

**設定方法：**

```bash
# 喺 TUI 入面
/think low
/think medium
/think high

# 或者 config 入面設預設
openclaw config set agents.default.think low
```

<Callout variant="tip">
**我嘅設定：** Crawler、Social 等自動化 Agent 用 `think: low`，Coder Agent 做複雜 review 先用 `think: high`。大部分日常任務 `think: low` 完全夠用。
</Callout>

## 招四：用 Cron Job 代替持續監控

新手常見錯誤：叫 Agent「幫我一直監控某個網站，有更新就通知我」。

問題係 Agent 會係咁跑、係咁問 AI，每次都傳入整個對話歷史，費用直線上升。

**正確做法：用 Cron Job**

```bash
# 設定每小時執行一次，用新 session
openclaw cron add --schedule "0 * * * *" \
  --agent crawler \
  --command "check site updates" \
  --new-session

# 每天早上 9 點跑 social agent
openclaw cron add --schedule "0 9 * * *" \
  --agent social \
  --command "daily social monitoring" \
  --new-session
```

每次 cron 執行都係一個新 session，token 消耗係最低嘅基礎量，唔會累積。

**節省效果估算：**

| 方式 | 一日 100 次監控嘅費用（Haiku） |
|---|---|
| 持續監控（長 session） | ~$2–5（context 不斷累積） |
| Cron Job（每次新 session） | ~$0.10–0.30 |

## 招五：監控用量 + 設上限

### 實時監控：TUI 底部

TUI 底部會顯示：
```
tokens 10k/272k (4%)  think low  claude-sonnet-4-6
```

- `10k`：當前 session 已用 token
- `272k`：context window 上限
- `(4%)`：使用百分比
- `think low`：當前 think mode

### 設用量上限（必做）

**OpenAI：**
1. 去 [platform.openai.com/settings/organization/limits](https://platform.openai.com/settings/organization/limits)
2. 設 **Soft limit**（達到時發 email）
3. 設 **Hard limit**（達到時停止 API call）

**Anthropic：**
1. 去 [console.anthropic.com/settings/limits](https://console.anthropic.com/settings/limits)
2. 設月度用量上限

**OpenClaw Config 設 Budget Alert：**

```bash
# 設每日 budget 上限（美元）
openclaw config set budget.daily_limit 5.00

# 設每月上限
openclaw config set budget.monthly_limit 50.00

# 達到上限嘅行為（warn / stop / fallback）
openclaw config set budget.on_limit warn
```

<Callout variant="warning">
**強烈建議：** 第一次設定 OpenClaw 就即刻喺 API provider 設 hard limit。唔係唔信任自己，係防止設定錯誤或者 runaway Agent 導致意外費用。
</Callout>

## FAQ

**Q：OpenClaw 本身使唔使錢？**

OpenClaw 軟件本身係開源免費。費用嚟自你用嘅 LLM API——即係每次 OpenClaw 問 Claude 或 GPT 問題時嘅 API call 費用。軟件免費，但跑 AI 嘅算力要錢。詳細睇[OpenClaw vs ChatGPT 費用比較](/compare/vs-chatgpt)。

**Q：用本地模型係咪完全免費？**

係，本地模型（例如透過 Ollama 跑 Llama、Mistral 等）唔需要 API 費用。但你需要有足夠嘅硬件：Apple Silicon Mac（M1/M2/M3/M4）或者有獨立 GPU 嘅 PC。模型質素亦通常低過頂級雲端模型。適合想完全本地化、唔想有任何費用嘅用戶。

**Q：點知我而家燒咗幾多錢？**

兩個地方睇：一係 TUI 底部嘅實時 token 顯示（`tokens X/Y (Z%)`），可以估算當前 session 嘅消耗。二係你嘅 API provider dashboard——OpenAI 有 Usage 頁面、Anthropic 有 Console 用量統計，都可以睇到實際費用。建議每隔幾日睇一次直至習慣咗自己嘅用量模式。

**Q：有冇方法設用量上限，用完就停？**

有。OpenAI 同 Anthropic 都支持喺 dashboard 設 hard limit，達到上限後 API call 會返回錯誤，OpenClaw 就唔能再發出請求。另外 OpenClaw config 入面都可以設 `budget.daily_limit`，達到後會根據你設嘅 `on_limit` 行為（warn / stop / fallback to cheaper model）處理。強烈建議兩個都設。
