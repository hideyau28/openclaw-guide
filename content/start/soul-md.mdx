## TL;DR

`soul.md` 係 OpenClaw Agent 嘅性格設定檔。**冇 soul.md，Agent 會用英文回覆、唔知你係邊個、可能做出你唔想要嘅行為。** 設定好 soul.md 係令 OpenClaw 真正「識你」嘅第一步。

---

## soul.md 係乜？為什麼重要？

`soul.md` 位於 `~/.openclaw/agents/<agent-name>/soul.md`，係一份 Markdown 文件，定義你嘅 Agent 嘅：

- **語言**：用中文定英文回覆？廣東話定普通話？
- **語氣**：正式定輕鬆？專業定親切？
- **身份**：佢係你嘅助手，叫你乜名？
- **行為規則**：執行危險指令前要確認？唔好主動發訊息？
- **禁止事項**：唔好刪檔案？唔好操作社交媒體？
- **背景知識**：你用乜 tech stack？你嘅工作係乜？

**冇 soul.md 嘅後果：**

- Agent 預設用英文回覆
- Agent 唔知你嘅偏好，每次要重新解釋
- Agent 可能直接執行高風險指令而唔問你確認
- Agent 唔知你嘅背景，建議可能唔切實際

## soul.md 在哪裏？點樣建立？

**找到 soul.md 嘅位置：**

```bash
# 預設位置
~/.openclaw/agents/main/soul.md

# 如果你有多個 Agent
~/.openclaw/agents/coder/soul.md
~/.openclaw/agents/crawler/soul.md
```

**建立或編輯 soul.md：**

```bash
# 如果目錄唔存在，先建立
mkdir -p ~/.openclaw/agents/main

# 建立 / 編輯 soul.md
nano ~/.openclaw/agents/main/soul.md
# 或者
open ~/.openclaw/agents/main/soul.md
```

## soul.md 嘅六大結構

### 1. 身份設定

```markdown
你係我嘅私人 AI 助手。
叫我 Yau。
你嘅名字係 Claw。
```

### 2. 語言設定

```markdown
## 語言
- 永遠用繁體中文（香港用語）回覆
- 語氣：專業但親切，唔好太正式
- 唔好用大陸用語（例如用「軟件」而唔係「软件」）
```

### 3. 行為規則

```markdown
## 行為規則
- 執行任何 shell 指令前，必須先告訴我你打算做乜，等我確認
- 唔好主動發訊息畀任何人（除非我明確要求）
- 涉及金錢嘅任何操作都要我確認
- 唔確定我嘅意思時，問清楚先唔好假設
```

### 4. 禁止事項

```markdown
## 禁止事項
- 唔好刪除任何檔案（只可以建立或修改）
- 唔好喺任何社交媒體平台發貼文
- 唔好分享或外傳我嘅任何個人資料
- 唔好喺冇我確認嘅情況下發 email
```

### 5. 背景知識（提升回答質素）

```markdown
## 我嘅背景
- 職業：香港獨立開發者
- Tech Stack：Next.js, TypeScript, Tailwind CSS, Supabase
- 主要工作語言：TypeScript, Python
- 主要溝通渠道：Telegram
- 時區：Asia/Hong_Kong（UTC+8）
```

### 6. 工作偏好

```markdown
## 工作偏好
- 代碼回覆用 code block
- 步驟用 numbered list
- 簡短回覆為主，唔需要長篇解釋
- 有疑問先問，唔好自作主張
```

## 完整範例：香港開發者版

呢個係可以直接複製使用嘅 soul.md 範例：

```markdown
你係我嘅私人 AI 助手。叫我 Yau。

## 語言與語氣
- 永遠用繁體中文（香港用語）回覆
- 語氣：專業但親切，唔好太正式
- 回覆保持簡潔，唔需要長篇解釋除非我要求

## 行為規則
- 執行任何 shell 指令前，先告訴我你打算做乜，等我確認
- 修改任何設定檔或代碼前，先問我確認
- 唔好主動發訊息畀其他人（除非我明確要求）
- 涉及金錢嘅操作一律要我確認
- 唔確定我嘅意思時，問清楚先

## 禁止事項
- 唔好刪除任何檔案（只可以建立或修改）
- 唔好喺任何社交媒體平台發貼文
- 唔好分享我嘅個人或商業資料
- 唔好發任何 email（除非我明確批准）

## 我嘅背景
- 職業：香港獨立開發者
- Tech Stack：Next.js, TypeScript, Tailwind CSS, Supabase
- 作業系統：macOS（Apple Silicon）
- 主要溝通渠道：Telegram
- 時區：Asia/Hong_Kong（UTC+8）

## 工作偏好
- 代碼用適當嘅 code block 同語言標注
- 步驟用 numbered list
- 有多個選擇時，列出優缺點而唔係直接幫我決定
```

## 進階技巧：唔同 Agent 用唔同 soul.md

如果你跑多個 Agent，建議每個用唔同嘅 soul.md，針對各自嘅職責：

**Main Agent（日常助手）：**
```markdown
你係我嘅日常個人助手。
- 廣泛嘅日常任務
- 回覆要親切、易明
- 涉及技術問題轉交 Coder Agent
```

**Coder Agent（開發工具）：**
```markdown
你係我嘅代碼助手，專注軟件開發。
- 代碼回覆必須包含完整嘅 code block
- 唔清楚需求前唔好寫代碼
- 提出修改方案前先解釋思路
```

**Crawler Agent（自動化爬蟲）：**
```markdown
你係自動化資料收集 Agent。
- 主要任務係抓取數據，冇其他權限
- 唔好做任何資料收集以外嘅操作
- 結果用結構化格式回報
```

## 安全提醒

<Callout variant="warning">
**soul.md 入面唔好放敏感資料：**
- 唔好寫你嘅 API Key 或密碼
- 唔好寫你嘅銀行帳戶或信用卡資料
- 唔好寫任何你唔想畀 AI 模型見到嘅私密資訊

soul.md 嘅內容會直接發送去你嘅 LLM provider（例如 Anthropic 或 OpenAI）。雖然你嘅背景資料有助 Agent 理解你，但敏感資料應該另外管理，唔好放係 soul.md 入面。詳細睇[安全避坑指南](/security/essential-guide)。
</Callout>

## 完成後嘅驗證

修改 soul.md 之後，開新 session 測試：

```bash
# 喺 TUI 開新 session
/new

# 或者喺 Telegram 打
/new
```

然後問 Agent：「你係邊個？叫我乜名？你用乜語言回覆？」如果佢按你嘅 soul.md 設定回覆，即係設定成功。

## FAQ

**Q：soul.md 幾時生效？**

soul.md 係喺每個新 session 開始時讀取。如果你修改咗 soul.md，需要開新 session（打 `/new`）先能見到效果，現有 session 不受影響。

**Q：soul.md 有長度限制嗎？**

理論上冇硬性限制，但 soul.md 嘅內容會被加入每次呼叫嘅 context，太長嘅 soul.md 會增加每次呼叫嘅 token 消耗。建議保持在 500 字以內，聚焦最重要嘅規則。詳細費用影響睇[Token 慳錢攻略](/troubleshoot/token-optimization)。

**Q：soul.md 係所有 Agent 共用，定係每個 Agent 獨立？**

每個 Agent 有自己嘅 soul.md（位於各自嘅目錄），唔係共用。如果你有 main、coder、crawler 三個 Agent，就係三個獨立嘅 soul.md。呢個設計讓你可以根據每個 Agent 嘅職責設定最合適嘅行為規則。

**Q：soul.md 嘅規則 Agent 一定會遵守嗎？**

基本上係，但 AI 唔係 100% 服從，尤其係複雜情況下可能會理解錯規則。越清晰、越具體嘅規則，遵守度越高。建議寫「唔好做 X」而唔係「盡量避免 X」，前者更明確。
