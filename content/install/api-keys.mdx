## TL;DR

OpenClaw 本身免費，但需要 LLM API Key 先可以用。**最快方法：OpenRouter 一個 key 用晒所有 model。**

| 想要乜 | 推薦 Provider |
|---------|---------------|
| 最快開始 | OpenAI Codex OAuth（免費、唔使信用卡） |
| 最慳錢 | DeepSeek 或 Ollama（本地免費） |
| 最高質素 | Claude Sonnet 4.6 或 GPT-5.4 |
| 最靈活 | OpenRouter（一個 key 用 400+ model） |

| Provider | 需信用卡 | 免費額度 | 主力模型 Input 價格 | 適合人群 |
|----------|----------|----------|---------------------|----------|
| OpenAI OAuth | 唔使 | 有 | 免費（OAuth 額度） | 新手首選 |

## 1. OpenAI（最多人用）

OpenAI 有兩種認證方式。新手強烈建議用 Codex OAuth。

### 方法一：Codex OAuth（推薦新手）

唔使信用卡、有免費額度、設定最簡單。

```bash
# 方法一：跑 onboard 自動引導
openclaw onboard
# 揀 OpenAI，瀏覽器彈出做 OAuth

# 方法二：直接登入
openclaw models auth login --provider openai-codex
```

瀏覽器彈出後，用 OpenAI 帳號登入，授權即完成。唔需要手動複製 Key。

### 方法二：API Key

需要綁信用卡，適合有重度用量、需要更高 rate limit 嘅用戶。

1. 去 `platform.openai.com` → API Keys → **Create new secret key**
2. 複製 Key（只顯示一次）
3. 設定：

```bash
openclaw models auth setup-token --provider openai
# 貼入你嘅 API Key
```

**可用模型：** `gpt-5.4`、`gpt-5.3-codex`、`gpt-5.2`、`gpt-4o-mini`

<Callout variant="warning">
**踩坑：** OAuth 同 API Key 係兩套獨立系統。OAuth 用 `--provider openai-codex`，API Key 用 `--provider openai`。搞混會出錯。
</Callout>

## 2. Anthropic（⚠️ 要小心）

Claude 系列模型質量高，但有重要警告。

1. 去 `console.anthropic.com` → API Keys → **Create Key**
2. 需要綁信用卡
3. 設定：

```bash
openclaw models auth setup-token --provider anthropic
# 貼入 API Key（格式：sk-ant-...）
```

**可用模型：** `claude-opus-4-6`、`claude-sonnet-4-6`、`claude-haiku-4-5`

<Callout variant="danger">
**帳號封禁風險：** OpenClaw 嘅使用方式（自動執行指令、持續運行 Agent、大量 API 呼叫）有機會觸發 Anthropic 嘅 Terms of Service 審查。已有用戶報告帳號被 ban。

建議：
- 用**獨立帳號**，唔好用你 Claude.ai 嘅同一個帳號
- 設月度用量上限
- 或者透過 **OpenRouter** 間接用 Claude model（更安全）
</Callout>

## 3. Google Gemini

有免費額度，適合想試唔想即刻付費嘅用戶。

1. 去 `aistudio.google.com` → **Get API Key**
2. 唔需要信用卡（有免費 tier）
3. 設定：

```bash
openclaw models auth setup-token --provider google
# 貼入 API Key（格式：AIza...）
```

**可用模型：** `gemini-2.5-pro`、`gemini-2.5-flash`

Gemini 2.5 Flash 係目前速度最快嘅模型之一，免費 tier 對新手試玩足夠。

## 4. OpenRouter（推薦 — 一個 Key 用晒所有 Model）

聚合平台，一個帳號存取所有主要 Provider 嘅模型。**香港直接用得到，唔使 VPN。**

1. 去 `openrouter.ai` → **Keys** → Create Key
2. 需要綁信用卡（充值後使用）
3. 設定：

```bash
openclaw models auth setup-token --provider openrouter
# 貼入 API Key（格式：sk-or-...）
```

設定完之後，你可以用任何 OpenRouter 支持嘅 model：

```bash
# 用 Claude（透過 OpenRouter）
openclaw config set agents.main.model anthropic/claude-sonnet-4-6

# 用 DeepSeek（透過 OpenRouter）
openclaw config set agents.crawler.model deepseek/deepseek-v3

# 用 GPT-5.4（透過 OpenRouter）
openclaw config set agents.coder.model openai/gpt-5.4
```

<AffiliateBanner provider="openrouter" />

**優點：**
- 一個 dashboard 睇晒所有用量
- 可以隨時換 model，唔使重新設 key
- 支援 400+ model
- 唔受地區限制，香港穩定可用
- 間接用 Anthropic model 可以降低被 ban 風險

**缺點：**
- 每次 API call 有少量加成（約 0–5%）
- 多一層中間商，latency 稍高

## 5. DeepSeek（最平之一）

價格係最大賣點，128K context window。

1. 去 `platform.deepseek.com` → API Keys → **Create API Key**
2. 唔需要信用卡（有少量免費額度）
3. 設定：

```bash
openclaw models auth setup-token --provider deepseek
# 貼入 API Key
```

**可用模型：** `deepseek-v3`、`deepseek-r1`

**優點：** Input token 價格係 Claude Sonnet 嘅 1/10，DeepSeek R1 有強推理能力。

**缺點：**
- Instruction-following 弱過 Claude / GPT
- 安全性較弱，更容易被 prompt injection
- 伺服器喺中國，延遲可能較高

<Callout variant="warning">
**安全提醒：** DeepSeek 係中國公司，數據可能受中國法律監管。如果你嘅 Agent 接觸敏感資料，建議用其他 Provider。詳細嘅安全考量睇[安全避坑指南](/security/essential-guide)。
</Callout>

## 6. 本地 Ollama（免費 + 完全離線）

唔想付任何 API 費用？唔想數據離開你部機？Ollama 係唯一方案。

### 安裝

```bash
# 安裝 Ollama
brew install ollama

# 啟動服務
ollama serve

# 拉取模型（選一個）
ollama pull llama3.2          # Meta Llama 3.2（輕量）
ollama pull mistral            # Mistral 7B
ollama pull qwen2.5-coder     # 代碼專用
```

### 設定 OpenClaw

```bash
openclaw config set agents.main.model ollama/llama3.2
openclaw config set model.ollama.base_url http://localhost:11434
```

### 硬件要求

| 模型大小 | 最低 RAM | 推薦 |
|----------|----------|------|
| 7B | 8GB | M2 Mac 16GB |
| 13B | 16GB | M2 Pro 16GB |
| 70B | 64GB | M3 Max 96GB |

<Callout variant="tip">
**Apple Silicon 用戶：** Ollama 喺 M 系列 Mac 跑得好流暢。M1/M2 16GB 跑 7B 模型完全夠用。
</Callout>

**優點：** 完全免費、完全離線、最高私隱。
**缺點：** 需要強硬件、推理速度慢、質素低過頂級雲端 model。

## Primary + Fallback 設定

可以設主要模型 + 後備模型，超 budget 或主 model 出問題時自動切換：

```yaml
# ~/.openclaw/config.yaml
agents:
  main:
    model: claude-sonnet-4-6         # 主要：質量好
    fallback_model: claude-haiku-4-5 # 後備：便宜
  coder:
    model: gpt-5.4                    # 主要：代碼強
    fallback_model: deepseek-v3      # 後備：平
  crawler:
    model: claude-haiku-4-5          # 抓數據唔需要貴 model
```

<Callout variant="warning">
**Fallback 安全考慮：** 唔好將 fallback 設為安全性太弱嘅 model。弱 model 更容易被 prompt injection——尤其 Crawler Agent 會接觸不受信任嘅網頁內容。詳細睇[安全避坑指南](/security/essential-guide)。
</Callout>

設定好 Key 之後，下一步係控制費用，睇 [Token 慳錢攻略](/troubleshoot/token-optimization)。

## FAQ

**Q：邊個 model 最平？**

雲端入面，GPT-4o mini（$0.15/1M input）同 DeepSeek V3（$0.27/1M）最平。完全免費就用 Ollama 本地 model 或者 OpenAI Codex OAuth 嘅免費額度。

**Q：可唔可以用免費 model？**

可以。OpenAI Codex OAuth 有免費額度（唔使信用卡），Google Gemini 有免費 tier，DeepSeek 有少量免費額度，Ollama 本地完全免費。

**Q：API Key 安全嗎？存喺邊？**

API Key 存喺你本機嘅 `~/.openclaw/` 目錄入面（加密儲存）。正常情況係安全嘅。主要風險係惡意 Skills——如果裝咗惡意 Skill，佢有機會讀取 API Key。所以裝 Skill 之前一定要做安全審查，睇[安全避坑指南](/security/essential-guide)。

**Q：可唔可以同時設幾個 Provider？**

可以。你可以為唔同 Agent 設唔同 Provider 同模型。用 `openclaw config set agents.<agent-name>.model <model-id>` 逐個設。

**Q：OpenRouter 同直接用 Provider 有乜分別？**

功能一樣。直連 Provider 冇加成、latency 低；OpenRouter 方便統一管理、一個 dashboard 睇晒用量，但有少量加成（0–5%）。如果你只用一兩個 Provider，直連更好；經常換 model 就用 OpenRouter。
