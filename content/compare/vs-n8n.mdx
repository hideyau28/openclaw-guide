| | OpenClaw | n8n | Make |
|---|---|---|---|
| **本質** | AI Agent（自然語言） | Workflow 自動化 | Visual Workflow Builder |
| **決策方式** | AI 自主判斷 | 你設計嘅流程 | 你設計嘅流程 |

## 三款工具嘅本質分別

### OpenClaw：AI 決策

你用自然語言告訴 OpenClaw 你想要什麼結果，AI 自己決定點執行。

```
你：「監控我嘅 GitHub repo，有新 PR 就幫我 review 同分類優先級」
OpenClaw：自主讀 PR → 理解代碼 → 撰寫 review → 發 Telegram 通知你
```

優勢係可以處理模糊指令、理解上下文、做出判斷。劣勢係唔確定性——AI 有時可能做出意料之外嘅決定。

### n8n：Workflow 設計

你用視覺化界面設計一個明確嘅流程圖，程式嚴格按流程執行。

```
Trigger（新 PR）→ 讀 PR 內容 → 發 Slack 通知 → 加 Label → 完成
```

優勢係確定性強，每次都按你設計嘅流程執行，唔會有意外。劣勢係唔能處理你冇預設過嘅情況，每個 case 都需要預先設計好。

### Make（前 Integromat）：Visual Builder

同 n8n 相似，但更注重易用性，大量預建模板，唔太懂技術嘅人也可以上手。

## OpenClaw 嘅優勢與劣勢

**優勢：**
- **自然語言設定**：唔需要學 workflow 概念，直接講你要乜
- **AI 判斷能力**：可以處理「如果 email 係緊急嘅就即刻通知，否則下午一次過摘要」呢類需要判斷嘅情況
- **靈活應對新情況**：冇預設過嘅情況，AI 也能嘗試處理
- **本地運行**：數據留係你自己部機

**劣勢：**
- **唔確定性**：AI 可能做出你唔想要嘅決定
- **費用較高**：每次呼叫都有 API 費用
- **安全風險**：本地 Agent 有完整系統權限，詳細睇[安全避坑指南](/security/essential-guide)
- **難以 debug**：AI 做錯咗，有時唔容易知道點解

---

## n8n 嘅優勢與劣勢

**優勢：**
- **確定性**：流程明確，每次都按你設計嘅方式執行
- **自建完全免費**：自己搭 n8n server 係 $0
- **社區大**：大量現成 workflow 模板
- **400+ 整合**：幾乎所有主流服務都有

**劣勢：**
- **需要學 workflow 概念**：有少量學習成本
- **唔能處理模糊任務**：每個情況都需要預先設計好
- **複雜 logic 難維護**：流程越複雜，維護成本越高

---

## Make 嘅優勢與劣勢

**優勢：**
- **最易用**：拖拉界面，唔識技術也可以上手
- **1500+ 整合**：整合數量最多
- **大量模板**：常見 workflow 基本上都有現成模板

**劣勢：**
- **付費**：免費 tier 有限，實際用量需要付費
- **閉源**：你嘅 workflow 數據存係 Make 嘅 server
- **同樣唔能處理模糊任務**：確定性工具嘅共同限制

## 邊種任務用邊款工具？

| 任務類型 | 推薦工具 | 原因 |
|----------|----------|------|
| 需要 AI 判斷嘅複雜任務 | OpenClaw | AI 自主決策 |
| 確定性嘅重複性流程 | n8n | 可靠、自建免費 |
| 快速設定簡單整合 | Make | 易用、多模板 |

| CRM 資料同步 | n8n / Make | 確定性流程 |
| 定時發社交媒體貼文 | n8n / Make | 確定性、唔需要判斷 |
| PR Code Review | OpenClaw | 需要理解代碼 |
| API Webhook 轉發 | n8n | 確定性、低成本 |

## 最佳實踐：OpenClaw + n8n 配合用

唔需要只揀一個工具。最強嘅組合係：

- **OpenClaw** 做需要 AI 判斷嘅部分（分析、理解、決策）
- **n8n** 做確定性嘅流程部分（數據同步、通知轉發、格式轉換）

例子：
```
n8n：監控新 GitHub PR（Webhook）→ 觸發 OpenClaw
OpenClaw：AI Review PR + 分類優先級
n8n：根據 OpenClaw 嘅結果 → 發通知到不同 Slack channel
```

呢個組合利用每款工具嘅強項，避免各自嘅弱點。

想知點設定 OpenClaw 同外部工具嘅整合，睇 [Skills 推薦指南](/skills/essential-setup)。

## FAQ

**Q：我完全唔識技術，應該揀邊個？**

Make 係門檻最低嘅，有大量模板，基本上拖拉就可以設定好簡單嘅自動化。如果你嘅需求係「Gmail 收到訂單 email 自動記入 Google Sheets」呢類確定性任務，Make 或 n8n 比 OpenClaw 更適合。OpenClaw 係給想要 AI Agent 嘅人，而唔係想要 workflow builder 嘅人。

**Q：OpenClaw 同 n8n 可唔可以整合？**

可以。n8n 有 HTTP Request node，可以呼叫 OpenClaw Gateway 嘅 API。反過來，OpenClaw 可以呼叫 n8n 嘅 webhook 觸發 n8n workflow。兩者整合嘅文檔仍然有限，需要自己摸索。

**Q：如果我只係想試自動化，應該先學邊個？**

建議由 Make 入門（易用、有免費 tier）。理解咗 workflow 概念之後，再決定要唔要升級去 n8n（更強、可自建）或者 OpenClaw（需要 AI 判斷嘅任務）。三者嘅概念有重疊，學咗一個之後其他兩個更易上手。
