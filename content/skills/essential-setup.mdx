## TL;DR

ClawHub 有 1800+ Skills，其中約 8% 被 Koi Security 標記為惡意。**1800 個裡面，呢 20 個最值得裝——但裝之前一定要做安全檢查。**

---

## 先搞清楚：Tools vs Skills 嘅分別

好多新手搞混呢兩個概念。弄清楚先，後面先睇得明。

**Tools = 權限開關**

決定 OpenClaw「可唔可以」做某件事。例如：
- `read` — 可唔可以讀你嘅檔案
- `write` — 可唔可以寫入檔案
- `exec` — 可唔可以執行 shell 指令
- `browser` — 可唔可以控制瀏覽器

**Skills = 教學手冊**

教 Agent「點樣」做某件事。例如：
- `obsidian` Skill 教 Agent 怎樣讀寫 Obsidian vault
- `github` Skill 教 Agent 怎樣操作 GitHub PR 同 Issues
- `telegram` Skill 教 Agent 怎樣收發 Telegram 訊息

**三個條件必須同時滿足，Skill 先能正常運作：**

```
Tools 權限開咗（exec / read / write）
         +
    Skill 裝咗（教 Agent 點用）
         +
  外部工具已安裝（例如 Obsidian app）
         ↓
      Skill 正常運作
```

**常見誤解：**
- 裝咗 `obsidian` Skill 但冇開 `write` Tool → Agent 識講點做，但實際做唔到
- 開咗 `exec` Tool 但冇裝任何 Skill → Agent 有權限，但唔知做乜
- 裝 Skill 唔會自動開 Tool 權限（這是安全設計）

```bash
# 安裝單個 Skill
openclaw skill install memory

# 睇已安裝嘅 Skills

# 搜尋 ClawHub
openclaw skill search obsidian

# 睇 Skill 詳情（安裝前先做呢步）
openclaw skill inspect <skill-id>
```

## 安全審查三步檢查法

ClawHub 有約 8% 嘅惡意 Skills（820+ 個，來源：Koi Security 報告）。**裝任何社區 Skill 之前，跟呢三步做。**

### 第一步：睇 Security Scan 結果

喺 ClawHub 頁面搵 Security Scan 一欄：

![ClawHub Skill inspect 頁面，顯示 Security Scan 結果](/images/articles/essential-guide/clawHub-skill-inspect.webp)

- **Benign + HIGH CONFIDENCE** = 相對安全，可以安裝
- **Benign + LOW CONFIDENCE** = 小心，建議睇源碼
- **冇 scan 結果** = 唔好裝，或者自己審查源碼先

### 第二步：檢查 Permission 要求

```bash
# 睇呢個 Skill 要求哪些權限
openclaw skill inspect <skill-id>
```

問自己：**呢個 Skill 嘅功能同佢要求嘅權限係咪合理？**

紅旗範例：
- 一個「天氣查詢」Skill 要求讀 `~/.ssh` 目錄 → 明顯可疑
- 一個「筆記管理」Skill 要求 `exec` 執行任意指令 → 不合理
- 一個「RSS 訂閱」Skill 要求網絡訪問 `*.analytics.io` → 可能係追蹤

### 第三步：檢查作者信譽

喺 ClawHub 睇作者頁面：

| 信號 | 說明 |
|------|------|
| 多個 Skills + 高 star + 有評論 | 較可信 |
| 只有一個 Skill + 零評價 + 下載量高 | 可疑 |
| 最近才創建帳號 + 只有一個 Skill | 可疑 |
| 源碼係 obfuscated（混淆） | 紅旗，唔好裝 |

## 安全設定（推薦）

```bash
# 只允許安裝官方認證嘅 Skills（最保守）
openclaw config set skills.verified_only true

# 關閉自動更新，改為手動審查
openclaw config set skills.auto_update false

# 確認設定
openclaw config get skills
```

<Callout variant="warning">
**自動更新有風險：** 一個原本安全嘅 Skill，作者可以喺更新版本加入惡意代碼。建議關閉自動更新，手動審查每次更新內容先安裝。詳細風險睇[安全避坑指南](/security/essential-guide)。
</Callout>

## FAQ

**Q：裝 Skill 會唔會自動開相應嘅 Tool 權限？**

唔會。Skills 同 Tools 係分開管理嘅。裝 `obsidian` Skill 唔會自動開 `write` Tool。呢個係安全設計——你需要明確決定畀哪些權限 Agent，唔係裝個 Skill 就自動獲得所有所需權限。如果 Skill 需要某個 Tool 但你冇開，Agent 會話佢你需要先開對應權限。

**Q：可唔可以自己寫 Skill？**

可以。Skill 本質上係一個 Markdown 格式嘅教學文件，教 Agent 點樣用某個工具或服務。你可以寫自己公司嘅內部工具 Skill、自己嘅工作流程 Skill，甚至上載去 ClawHub 分享。自己寫嘅 Skill 只存係本地，唔需要安裝，直接放喺 `~/.openclaw/skills/` 目錄即可。

**Q：裝咗幾多 Skills 先算「太多」？**

質比量重要。我建議裝少少但用緊嘅，唔好見到有趣就裝。每個 Skill 都可能帶嚟安全風險（尤其是社區 Skill），而且太多 Skill 會增加 context window 佔用，影響 Agent 判斷。我自己只裝我實際每星期都用到嘅 Skills，目前係 8 個左右。

**Q：ClawHub 上邊啲 Skill 千祈唔好裝？**

- 冇 Security Scan 結果嘅社區 Skill（尤其係下載量高但冇評論）
- 要求跟功能唔相符嘅過份權限嘅 Skill
- 源碼係 obfuscated（混淆過）嘅 Skill
- 只有一個 Skill 嘅新帳號作者發布嘅 Skill
- 要求讀取 `~/.ssh`、`~/.aws`、`~/Library/Keychains` 等敏感目錄嘅 Skill

**Q：點樣移除已裝嘅 Skill？**

```bash
# 移除單個 Skill
openclaw skill uninstall <skill-id>

# 例如
openclaw skill uninstall notion

# 確認已移除
openclaw skill list
```

移除 Skill 唔會改變你嘅 Tool 權限設定，只係移除那份「教學手冊」。
